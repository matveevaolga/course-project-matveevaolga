# Security Policy

## Для курса
- Пожалуйста, **не** храните ключи/секреты в репозитории.
- Сообщайте об уязвимостях преподавателю/ТА через приватный канал.
- В коде используйте единый формат ошибок (см. README).
- Во время курса используйте **синтетические** данные (без ПДн/платежей).

## Для проекта Feature Votes

### Контакты и SLA
- Сообщайте об уязвимостях преподавателю/ТА через приватный канал
- **SLA:** Ответ в течение 2 дней, фикс в течение 3 дней
- Контакты: через приватный канал
- **Приоритет обработки**: высокий для уязвимостей голосования

### Требования безопасности для Feature Votes
- К голосованию допущены только **аутентифицированные пользователи**
- **Не** храните ключи/секреты в репозитории
- Валидация входных данных фич и голосов
- Реализуйте аутентификацию для защиты от дублирования голосов (один пользователь - один голос за фичу)
- Не используйте реальные персональные данные в тестовой среде
- Для тестирования используйте синтетических пользователей и тестовые фичи
- **Шифрование паролей** пользователей с использованием bcrypt/scrypt
- Создатель фичи может редактировать и удалять **только свои фичи**
- Защита от манипуляций с результатами голосования
