# BDD Scenarios for Feature Votes NFR

## Сценарий 1: Быстрое голосование
  Feature: Производительность голосования
  Scenario: Пользователь голосует за фичу
  Given система работает под нагрузкой 500 пользователей
  When пользователь отправляет POST /features/1/vote
  Then время ответа должно быть ≤ 200 мс
  And голос должен быть сохранен в системе

## Сценарий 2: Защита от накруток
  Feature: Безопасность голосования
  Scenario: Попытка дублирующего голосования
  Given пользователь уже голосовал за фичу 1
  When он пытается проголосовать повторно
  Then система возвращает ошибку 409 Conflict
  And счетчик голосов не изменяется

## Сценарий 3: Доступность системы
  Feature: Надежность системы
  Scenario: Проверка доступности в пиковое время
  Given пиковая нагрузка 1000 RPS
  When пользователь запрашивает GET /health
  Then система возвращает статус 200 OK
  And время ответа ≤ 100 мс

## Сценарий 4: Целостность данных
  Feature: Надежность хранения
  Scenario: Восстановление после сбоя
  Given произошел сбой базы данных
  When система перезапускается
  Then все голоса должны быть восстановлены
  And статистика должна остаться корректной
